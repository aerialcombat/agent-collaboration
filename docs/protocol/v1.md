# Peer-inbox wire protocol, v1

This directory holds the language-neutral spec for the agent-collaboration
peer-inbox wire protocol. `v1.proto` is the authoritative artifact; this
document describes the versioning rules, the framing layer, and the
compatibility guarantees the spec commits to.

## Status

- **v1** — shipped in v3.0 of this repo (W1 of the Go rewrite).
- Exercised over Unix socket from day one; cloud transport (gRPC) will reuse
  the same `Envelope` unchanged when v2 ships.

## Framing on the Unix socket

Every frame on the wire is:

```
[4 bytes: magic = 0xA9 0xC0 0x1A 0xB0]
[4 bytes: big-endian uint32 length of the serialized Envelope]
[length bytes: serialized Envelope (protobuf)]
```

Magic is the first four bytes `0xA9 0xC0 0x1A 0xB0` ("agent-collab" leetspeak,
distinctive enough to reject legacy JSON with a loud error). The length is
big-endian for easier hex-dump debugging.

Messages exceeding 64 KiB on the wire are rejected with an `Error` frame
code `frame_too_large`. (The `body` field in `PeerSend` / `PeerBroadcast`
has its own 8 KiB cap; the larger outer limit leaves headroom for
metadata-heavy frames like `PeerReceiveResponse` that bundle multiple
`InboxMessage` entries.)

## Compatibility rules

### Additive-only

- Never renumber fields.
- Never change a field's type.
- New fields MUST be added at unused numbers. Old binaries ignore them.
- Removed fields reserve their number via `reserved N;` so new fields do
  not accidentally reuse the slot.

### Capability negotiation

First frame on every new connection is a `Hello` from each side. Both sides
exchange before any other message:

- `Hello.version` — mismatch → send an `Error { code: "version_mismatch" }`
  and close the connection. v1 only accepts v1.
- `Hello.supported_types` — advertise the payload type names this peer
  handles. The sender SHOULD skip types the receiver did not advertise.
  Receivers MUST drop (log + ignore) unknown `Envelope.type` values.

### Forward-compat rule

Receivers do NOT crash on unknown payload types. They log a structured
`slog` event with `{event: "unknown_envelope_type", type: ..., peer: ...}`
and drop the frame. This allows newer senders to ship new types to older
receivers without breaking them.

## Breaking-change protocol

If a breaking change is unavoidable:

1. Create `v2.proto` as a sibling file; do not edit `v1.proto`.
2. Bump `Envelope.version` to `2` on the new path.
3. Advertise v2 in `Hello.supported_types`; negotiate down to v1 when
   connecting to an older peer.
4. Deprecate v1 with a dated sunset in CHANGELOG.md. Minimum 6-month overlap.

## Wire formats by payload type

| Payload type                    | Purpose                            | Direction        |
|---------------------------------|------------------------------------|------------------|
| `agentcollab.v1.Hello`          | Capability handshake               | Both, on connect |
| `agentcollab.v1.PeerSend`       | Unicast message                    | Sender → Peer    |
| `agentcollab.v1.PeerBroadcast`  | Fan-out message                    | Sender → Room    |
| `agentcollab.v1.PeerReceiveRequest`  | Pull unread (hook path)       | Receiver → Local |
| `agentcollab.v1.PeerReceiveResponse` | Unread messages               | Local → Receiver |
| `agentcollab.v1.SessionRegister`     | Register (cwd, label) pair    | CLI → Local      |
| `agentcollab.v1.Error`               | Structured failure frame      | Any → Peer       |

## Error diagnostics

`Error` frames carry enough context to debug wire issues without a packet
capture:

- `code`: string with documented valid values (see `v1.proto`). Enum conversion deferred to `v2.proto` under the additive-only discipline.
- `frame_size_bytes`, `first_bytes_hex`, `expected_magic_hex`: surface the
  raw bytes that triggered the rejection so post-rollout debugging reads a
  `slog` event, not a tcpdump.

## Generated code

Go bindings live at `go/pkg/protocol/`. Regenerate via
`scripts/build-proto.sh`. Generated code is committed to the repo
(per the scope doc) so the build doesn't depend on `protoc` being
installed on every developer machine.

## Testing

- `tests/protobuf-roundtrip.sh` (W3 DoD) — assert Python ↔ Go byte-for-byte
  roundtrip equivalence for every payload type declared here.
- `tests/interop.sh` (W2) — assert the 4-cell parity matrix holds
  (Go ↔ Python, both directions, with the magic-byte sniffer on the Python
  listener).
