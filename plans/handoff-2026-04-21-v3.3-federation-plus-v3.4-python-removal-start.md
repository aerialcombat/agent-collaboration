## Architectural decisions ratified this session

1. **Room IDs are `<host>:<pair_key>`.** `home_host` is immutable once stamped. Unreachable home host = unreachable room, full stop. No cross-host replication, no central registry, no split-brain — those are explicit non-goals (see `v3.3-symmetric-federation-scoping.md` §2).
2. **Oozoo integration deferred.** Business panel (six experts) unanimously flagged oozoo-drives-federation-architecture as premature. Build a throwaway mobile-chat bridge when validating the job, not sooner.
3. **Python CLI gets deleted, not coexists.** Owner ratified full removal — "ok with full removal as long as we can test them." Phase 0's parity harness is the "as long as we can test them" clause made concrete.
4. **Go peer-web talks native to its SQLite store.** No more sharing business logic across language boundaries via shell-outs. Python CLI remains the operator surface until Phase 5 flips it.

## Tech debt worth knowing

- **`peer-inbox` Go binary is tiny today** — only handles `daemon-claim`/`daemon-complete`/`daemon-sweep`/`daemon-reset-session`. Phase 2+3 rewrites convert it into the full CLI. Package layout TBD: the v3.4 scoping doc mentions `go/pkg/peerinbox/` but Phase 1 put the logic in `go/pkg/store/sqlite/send.go` pragmatically. Phase 3 should consider the package move if the CLI imports pull the logic out cleanly.
- **`MaxOpenConns=1` deadlock risk.** SQLiteLocal is single-conn. Any write path that grabs `db.Conn()` must release it before calling another method on `s.db`. Phase 1 had one bug here (fixed in `4068bfa`; see `releaseConn()` pattern in `send.go`). Replicate the pattern in Phase 3 write verbs.
- **Broadcast response shape is new.** `/api/send` with `to=""` or `to="@room"` now returns `{broadcast: true, recipients: [...]}` instead of the pre-v3.4 stdout-echo shape. If any client depended on the old opaque shape, it breaks — but the web UI's composer is the only consumer today.
- **Federation smoke test leaks channel pushes.** The test registers `laptop_sender` against a process-tree socket that happens to be this Claude session's own peer-inbox MCP listener, so real channel pushes arrive during test runs. Cosmetic — test passes — but future Phase 3 / CLI smoke tests should bind mock sockets.
- **v3.4 Phase 0 harness has no mock channel-push listener yet.** Tests that exercise send paths can't verify push behavior deterministically until one is added (plan §6 risk register notes this).
