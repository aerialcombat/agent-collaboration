## 4. Per-item impl surfaces

Coder's 2026-04-19 spot-probe (symbol-keyed refs per §13 rot discipline) confirms:

**Item 1 — β-i #3 stale-marker auto-reaper on register collision.**
- Surface: register path in `scripts/peer-inbox-db.py` (session-register handler).
- Today: `STALE_THRESHOLD_SECS=30min` constant exists; no threshold-based auto-reaper fires on register collision.
- Change: on register collision (same label, same cwd), check existing row's `last_seen_at`; if older than `STALE_THRESHOLD_SECS` AND no live socket, auto-reap (delete stale row) before inserting new registration. Surface a "reaped stale registration" log line.
- Go parity: if `agent-collab session register` Go path also handles collision, extend there.
- Test: add subtest to existing register-path test OR standalone `tests/session-register-reaper.sh` covering (a) fresh collision → error preserved, (b) stale collision → reap + new row.

**Item 2 — β-i #4 session-key → label disambig auto-infer `--as`.**
- Surface: session-key resolver in `scripts/peer-inbox-db.py` (`resolve_self` + `SESSION_KEY_ENV_CANDIDATES`).
- Today: key-based path ships; gap is "key set but no marker by hash" fall-through — when env var is set but no registered session matches the key hash, resolver doesn't fall back to label-based disambig.
- Change: when unique env var identifies a session AND that session is registered in the current cwd, infer `--as <label>` without requiring the operator to pass it. When env var is absent OR multiple sessions match, keep existing "multiple sessions registered; pass --as" error.
- Test: extend existing resolver test OR standalone `tests/session-key-disambig.sh` covering (a) env set + unique match → inferred label, (b) env absent + multiple sessions → existing error, (c) env set + no match → existing error.

**Item 3 — β-i #5a `peer status --pair-key <K>` cheap core.**
- Surface: new subcommand `peer status` in `scripts/peer-inbox-db.py`; parses `--pair-key` flag; queries `sessions` (roster + roles + last_seen_at + turn_count via existing `_fetch_rooms` extension) + `inbox` (unread count per peer).
- Output: plain-text table (label, agent, role, last_seen_at, turn_count, unread). Machine-readable mode via `--json` optional for v1 but recommended for future `peer-inbox-web` consumption.
- Deferred (5b per brainstorm): round-aware fields (current round + who-has-broadcast) — requires transcript parsing; ship as heuristic-flagged output in v4+ typed-payload era, NOT in this cluster.
- Test: `tests/peer-status-pair-key.sh` — seed test DB with 3 registered peers + 2 unread inbox rows; invoke; assert columns present + counts correct.

**Item 4 — γ legacy `init_db()` removal.**
- Surface: `def init_db(path: Path) -> None:` in `scripts/peer-inbox-db.py` (symbol-keyed; line drifts). Caller at the same file's DB-opening path (symbol `cmd_` dispatch or wherever `init_db` is invoked — verify at impl time).
- Change: delete `def init_db` + its call-site. DB init now always flows through the goose-owned baseline + migrate binary path (W1 `6568f8b` + Topic 3 `c96868f`).
- Prereq: confirm all code paths that relied on `init_db` for fresh-DB bootstrap now route through the goose migrate path. If any residual path exists, that path migrates first.
- Gate: `tests/migrate-upgrade-path.sh` — new test verifying (a) fresh DB at schema version 0 → `peer-inbox-migrate up` reaches current version, (b) DB at an arbitrary pre-A3 schema version → migrate up succeeds without `init_db` fall-back, (c) double-migrate is idempotent.
