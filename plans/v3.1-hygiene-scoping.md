## 4. Per-item impl surfaces

Coder's 2026-04-19 spot-probe + round-1 reviewer returns (gamma + reason + live-context lanes) shape the surfaces below. All refs symbol-keyed per §13 rot discipline (v1's line-number hedges removed per gamma T1 + reason H5).

**Item 1 — β-i #3 stale-marker auto-reaper on register collision.**
- Surface: register handler in `scripts/peer-inbox-db.py` (cmd-dispatch for `session register`).
- Today: `STALE_THRESHOLD_SECS=30min` constant exists; no threshold-based auto-reaper fires on register collision.
- Change: on register collision (same label, same cwd), check existing row's `last_seen_at`; if older than `STALE_THRESHOLD_SECS` AND no live socket, auto-reap (delete stale row) before inserting new registration. Surface a `event=session.register.reaped` log line naming the displaced session-key hash.
- Go parity: if `agent-collab session register` Go path also handles collision, extend there. v2 gate asserts Go/Python parity OR explicitly documents Python-only scope.
- Test: `tests/session-register-reaper.sh` subtests: (a) fresh collision → error preserved, (b) stale-and-no-socket collision → reap + new row, (c) **stale-but-socket-live collision → DO NOT REAP** (false-positive guard), (d) Go/Python parity OR Python-only documented.

**Item 2 — β-i #4 session-key → label disambig fall-through.**
- Surface: session-key resolver in `scripts/peer-inbox-db.py` (`resolve_self` + `find_marker_by_key` + `_verify_and_return_marker` + `SESSION_KEY_ENV_CANDIDATES`).
- Today: happy-path works — env var set + unique hash match → label inferred via `find_marker_by_key → _verify_and_return_marker`. **Real friction** (reproduced this session + coder round-1 #6): env var set + hash mismatch + multiple markers in cwd — resolver falls through to the generic "multiple sessions registered; pass --as" error, losing the env-var signal entirely.
- Change: when env var is set AND hash mismatches all markers AND multiple markers exist in cwd, retry resolution using the env-var label-hint (e.g. `CLAUDE_SESSION_ID` value hint) against registered labels; if unique label match exists, infer. If still ambiguous, keep existing error but include env-var diagnostic (`env var AGENT_COLLAB_SESSION_KEY=<hash-prefix> did not match any registered session; hints=<...>`).
- Test: `tests/session-key-disambig.sh` subtests: (a) env set + unique hash match → label inferred (happy-path regression), (b) env absent + multiple sessions → existing error, (c) env set + no hash match → existing error, (d) env set + hash mismatch + label-hint unique → label inferred, (e) env set + hash mismatch + label-hint ambiguous → diagnostic error.

**Item 3 — β-i #5a `peer status --pair-key <K>` cheap core.**
- Surface: new subcommand `peer status` in `scripts/peer-inbox-db.py` (dispatch through existing `cmd_peer_*` pattern); parses `--pair-key` flag; queries `sessions` (roster + roles + `last_seen_at` + `turn_count` via extension of `_fetch_rooms`) + `inbox` (unread count per peer: `SELECT COUNT(*) FROM inbox WHERE to_label=? AND claimed_at IS NULL GROUP BY to_label`).
- Output: plain-text table (label, agent, role, last_seen_at, turn_count, unread). `--json` machine-readable mode optional for v1 but recommended for future `peer-inbox-web` consumption.
- Deferred (5b per brainstorm): round-aware fields (current round + who-has-broadcast) — requires transcript parsing; ship as heuristic-flagged output in v4+ typed-payload era, NOT this cluster.
- Test: `tests/peer-status-pair-key.sh` — seed test DB with 3 registered peers + 2 unread inbox rows; invoke `peer status --pair-key <K>`; assert 3 rows + expected columns + correct unread counts.

**Item 4 — γ legacy `init_db()` removal.**
