## v1.7 goal: "install agent-collab" as one sentence

The developer UX today still requires: install script → edit
`.mcp.json` → launch claude with `--dangerously-load-development-channels
server:peer-inbox` → register per session → share cwd implicitly. Too
many steps; each is a drop-off point.

Target UX (what DJ specified 2026-04-17):

```
User: "install agent-collab"
Agent: [clones → installs → adds .mcp.json → drops slash commands → verifies → done]

(new terminal, claude session 1)
User: /agent-collab
Agent: Label?  >> backend
Agent: Registered as 'backend'. Pair key: swift-canyon-8f3a

(new terminal, claude session 2)
User: /agent-collab
Agent: Label?  >> frontend
Agent: Join an existing pair or new?  >> join
Agent: Paste pair key: swift-canyon-8f3a
Agent: Joined. Paired with 'backend'.

User (from either): /peer tell <other> the TTL changed to 15m
Agent: Sent.
```

## v1.7 scope

Ordered dependencies:

1. **Pair-key schema + register/send changes** (foundation, ~2h)
   - Add `sessions.pair_key` column (idempotent migration).
   - `session register --pair-key KEY` joins the room. If no key and
     no existing key in DB for this cwd, generate a three-word slug
     (`adjective-noun-XXXX` or similar).
   - `peer send --to X` resolves X by `(pair_key, label)` if a key
     is set on the caller; falls back to `(cwd, label)` otherwise.
     Additive — don't break existing cwd-scoped flows.
   - `peer list` scoped by pair_key when set.
   - Schema: new `rooms` table optional; could start with just the
     `pair_key` column + `labels-in-same-pair_key` as the "room."

2. **Slash commands** (~1h)
   - `commands/agent-collab.md` — interactive: prompts for label,
     asks new-or-join, collects pair key on join.
   - `commands/peer.md` — short form: `/peer send <to> "..."`,
     `/peer check`, `/peer end`.
   - Installer drops these into `~/.claude/commands/` (idempotent,
     backed up on update).

3. **Install skill** (~1h)
   - `skills/install-agent-collab.md` — a Claude skill that runs
     when the user says "install agent-collab". Skill does: check if
     repo already cloned; if not, `git clone` to
     `~/Development/agent-collaboration`; run
     `./scripts/install-global-protocol`; verify via doctor; print
     "ready, try /agent-collab".
   - Alternative: one-liner bash installer `curl -sSL
     https://.../install | bash` that shells this without an agent.

4. **Auto-label + key generation** (~30min)
   - Three-word slug generator (human-memorable, low collision).
     `swift-canyon-8f3a`-shaped. Source word list inline — small list
     OK for v1.7.
   - If user doesn't specify `--label`, generate one (but prompt to
     confirm in the slash command).

5. **Auto-inference for send/receive** (~30min)
   - With exactly one peer in current pair-key, `peer send "..."`
     without `--to` infers the recipient. Similar for `peer receive`.

6. **Test cross-runtime** (~1h)
   - Launch Claude + Codex in same pair. Verify:
     (a) Codex can run `agent-collab session register --pair-key ...`
     via its shell/exec tool.
     (b) Messages flow both ways via the hook path (Codex doesn't
     have `UserPromptSubmit`, so it polls via `peer receive` in its
     own turn).
   - Document whatever works and whatever doesn't.
   - Same test with Claude + Gemini if time permits.

7. **Smoke test additions** (~30min)
   - Pair-key-scoped send/receive scenario.
   - Slug generator determinism (same input → same slug) + collision
     rarity.
   - `/agent-collab` flow exercised via scripting (simulate the prompts).

## Design caveats

- **Keep cwd-scoping as fallback.** Don't remove it. Users who launch
  both sessions in the same repo without bothering with pair keys
  should still work.
- **Slash commands must fail open.** If `agent-collab` isn't on PATH
  or the helper errors, the slash command prints a clear "install
  first" message, not a stack trace.
- **Install skill assumes `git` + network.** Document fallback
  (manual clone) for air-gapped dev machines.
- **Three-word slug wordlist** — use ~128 adjectives + 128 nouns to
  stay compact but give 16K combos + 4-hex suffix for 16^4 suffix
  combinations. Unlikely collision in realistic use.

## What NOT to do in v1.7

- No group chats (3+ labels in one pair-key). Current plan keeps
  pair-key semantics to 2 labels for simplicity. Group chats are v2.0.
- No NL-spawn (Agent Teams style). Overkill.
- No cross-machine. Still same-machine SQLite.
- No archive retention. Unchanged from v1.6.
- No Slack-UI polish (search, keyboard shortcuts, reaction emoji,
  threads). Those are v2.0+.
